Plano de Inserção de Dados (Ordem das Tabelas)
Esta é a ordem lógica que o seu script Python deve seguir para inserir os dados, respeitando todas as dependências:

Departamentos: Gere e insira todos os departamentos primeiro, mas com a coluna ID_gerente definida como NULL para todos. Neste momento, não existem funcionários para serem gerentes.
Script SQL gerado: INSERT INTO Departamentos (ID_depart, nome, ID_gerente) VALUES (1, 'Recursos Humanos', NULL);

Funcionarios: Agora que os departamentos existem, gere e insira os 1000 funcionários. Para cada funcionário, pode atribuir um ID_depart aleatório de um dos departamentos criados no passo 1.
Script SQL gerado: INSERT INTO Funcionarios (ID_fun, NIF, ..., ID_depart) VALUES (101, '...', ..., 1);

UPDATE Departamentos (Fechando o Ciclo): Agora que todos os funcionários existem, volte à tabela Departamentos. Para cada departamento, escolha um funcionário aleatório (preferencialmente um que pertença a esse mesmo departamento) e defina-o como gerente.
Script SQL gerado: UPDATE Departamentos SET ID_gerente = 101 WHERE ID_depart = 1;

Entidades que dependem apenas de Funcionarios: A ordem entre estas não importa. Pode gerar e inserir dados para:
Remuneracoes (e as suas especializações Salario e Beneficios)
Dependentes
Ferias
Faltas
Historico_empresas

Avaliacoes: Para esta tabela, precisa que tanto o ID_Fun (avaliado) como o ID_avaliador já existam na tabela Funcionarios. Pode escolher aleatoriamente dois IDs de funcionário distintos para cada registo de avaliação.

Utilizadores e Permissoes:
Primeiro, crie os Utilizadores, escolhendo um subconjunto dos seus funcionários.
Depois, para cada utilizador criado, adicione uma ou mais entradas na tabela Permissoes.

Módulo de Recrutamento: Estas tabelas são maioritariamente independentes do resto, mas Candidato_a depende de Funcionarios.
Candidatos
Vagas (pode ligá-las a Departamentos existentes)
Requisitos_vaga (depende de Vagas)
Candidato_a (depende de Candidatos, Vagas e Funcionarios para o ID_recrutador).

Módulo de Formações:
Formacoes
Teve_Formacao (depende de Formacoes e Funcionarios).
